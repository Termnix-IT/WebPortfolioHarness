# Checklists

経歴ページ改善時に使うチェックリストです。

## Data Update Checklist

- [ ] `data/career.md` に新しい担当領域を追記した
- [ ] `data/skills.md` に経験根拠を追記した
- [ ] `data/projects.md` に実装状況を追記した
- [ ] `data/certifications.md` の資格状態を更新した
- [ ] `data/posts.md` に記事URLまたは公開予定を追記した
- [ ] `data/links.md` の公開URLを確認した

## Public Repository Checklist

- [ ] 住所、電話番号、非公開メールアドレスがない
- [ ] 顧客名、案件名、契約情報がない
- [ ] 内部IP、認証情報、API key、tokenがない
- [ ] 業務固有の障害詳細を書いていない
- [ ] 実務経験と個人検証が分かれている
- [ ] 学習中の内容を取得済み・実務済みとして書いていない

## Portfolio Update Checklist

- [ ] 反映元の `data/` が最新である
- [ ] 反映案が `output/proposals/` にある
- [ ] 文案が事実と一致している
- [ ] モバイル表示で長すぎない
- [ ] 既存デザインの文量に合っている
- [ ] 既存ポートフォリオrepoで表示確認した

## Review Checklist

- [ ] 強みが具体的に伝わる
- [ ] 運用保守、監視、障害切り分け、自動化の文脈がある
- [ ] プロジェクト説明に目的と運用観点がある
- [ ] 資格・学習中・公開準備中の状態が明確
- [ ] 次に直す箇所が3件以内に絞られている
