# Workflow

このリポジトリを使って、経歴ページを継続的に改善するための運用フローです。

## Basic Flow

1. `data/` を更新する
2. `prompts/` を使ってAIにレビューや文案生成を依頼する
3. `templates/` の形式で提案を整理する
4. 生成案を人間が確認する
5. 問題がなければ既存ポートフォリオrepoへ手動反映する

## Update Timing

- 新しい職務経験や担当領域が増えたとき
- 資格を取得したとき
- Qiita記事やGitHub成果物を公開したとき
- ポートフォリオページの見せ方を変えたいとき
- 転職活動向けに訴求軸を見直すとき

## Manual Review Checklist

- [ ] 公開してよい情報だけが書かれている
- [ ] 実務経験と個人検証が区別されている
- [ ] 事実と異なる表現がない
- [ ] 誇張した表現がない
- [ ] インフラエンジニアとしての強みが伝わる
- [ ] 既存ポートフォリオrepoのデザインに合う文量になっている

## Output Handling

`output/` は一時的な生成物を置く場所です。正式な情報源は `data/` です。

生成案を残したい場合は、個人情報や機密情報が含まれていないことを確認してから保存してください。
