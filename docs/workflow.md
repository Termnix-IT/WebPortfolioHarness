# Workflow

このリポジトリを使って、経歴ページを継続的に改善するための運用フローです。

## Basic Flow

1. `data/` を更新する
2. `prompts/` を使ってAIにレビューや文案生成を依頼する
3. `templates/` の形式で提案を整理する
4. 生成案を人間が確認する
5. 明示的な許可が出た場合だけ、既存ポートフォリオrepoへ反映する

## Approval Flow

既存ポートフォリオrepoへ反映する前に、必ず反映案を確認します。

1. 変更したい内容を `output/proposals/` などに整理する
2. 変更対象ファイル、対象セクション、差し替え文案を提示する
3. 事実、公開可否、文量、既存デザインとの相性を確認する
4. ユーザーが明示的に許可した場合だけ、既存ポートフォリオrepoへ反映する
5. 反映後にデスクトップ表示とモバイル表示を確認する

## Update Timing

- 新しい職務経験や担当領域が増えたとき
- 資格を取得したとき
- Qiita記事やGitHub成果物を公開したとき
- ポートフォリオページの見せ方を変えたいとき
- 転職活動向けに訴求軸を見直すとき

## Manual Review Checklist

- [ ] 反映案を事前に提示している
- [ ] ユーザーから明示的な反映許可が出ている
- [ ] 公開してよい情報だけが書かれている
- [ ] 実務経験と個人検証が区別されている
- [ ] 事実と異なる表現がない
- [ ] 誇張した表現がない
- [ ] インフラエンジニアとしての強みが伝わる
- [ ] 既存ポートフォリオrepoのデザインに合う文量になっている

## Output Handling

`output/` は一時的な生成物を置く場所です。正式な情報源は `data/` です。

生成案を残したい場合は、個人情報や機密情報が含まれていないことを確認してから保存してください。
